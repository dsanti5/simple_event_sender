Responses

There are three types of respones cases from the server to the client that are possible.

Response case 1: Successfully created event

The response consists of 2 parts:

  • status : Status value 200
  • json: Sends json object body that contains two fields “message” and “event”, where “message” contains a message that informs about that the event was successfully created in text as "Successfully created event". The json object is constructed as follows:
    • {message: "Successfully created event", event : event}

Response case 2: Authorization successful

The response consists of 2 parts:

  • status : Status value 200
  • json: Sends json sting to the body containing the message “Authorization successful”.

Response case 3: EventSenderErrorCode

The response consists of 3 parts:

  • status : Status code 403
  • header: Contains a specific error code with the field "EventSenderErrorCode" that can possess any of the error codes in the table below
  • json: Sends json object body that contains two fields “Error Type” and “Error message”, where “Error Type” are specified in Table 1 and “Error Message” contains a message that is generated where the validation throws an error, that specifies relevant information the validation error. The json object is constructed as follows:
    • {"Error Type" : exception.errorType, "Error Message" : exception.message}

Error codes

Error Code Error Situation
EventSenderException [1000]
INVALID_OBJECT_CREATED An invalid object was created
AuthorizationException [1100]
BAD_PASSWORD User entered wrong password
BAD_USERNAME Username not found in database
BAD_TOKEN Token missing or illegal token
RabbitMQException [1200]
CONNECTION_ERROR Token missing or illegal token
CHANNEL_ERROR Problem with creating RabbitMQ channel
EiffelException [1300]
INVALID_PARAMETER_OBJECT Parameter object does not match parameter schema
UNSUPPORTED_EDITION Not latest edition from client
INVALID_SCHEMA Occurs when node.js schema validator (ajv) detects an invalid schema
MISSING_LINKS Required links missing
ILLEGAL_LINK_TARGET Link is corresponding to illegal target
UUID_NOT_UNIQUE The same UUID found multiple times during query
LINK_MATCH_NOT_FOUND No legal targets corresponding to target in database
MULTIPLE_LINKS_FOUND Multiple possible links for link. Eiffel standard rejects multiple links for the event meta type. AllowSeveralMatches is false for link
DatabaseException [1400]
DUPLICATE_UUID_ERROR The sent events UUID is already in use by another event in the DB
LookupException [1500]
