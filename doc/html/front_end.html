Front-end

On this page information regarding using the web-based event sender can be found.

Using the web-based event sender includes three main functions

  1. Logging into the system
  2. Submiting an event
  3. Logging out of the system

Logging in

When pressing the Log in button, the user will be directed to a new page where the user needs to enter his/hers username and password before being able to submit any new events. When the user is logged in to the system, the Log in button will no longer be visible.

Username

In the field titled Username enter your personal username given to you by the system administrator, remember that your username is case-sensitive.

Password

In the field titled Password enter your personal password, remember that your password is case-sensitive.

Error messages

If the combination of username and password is not valid an error message will appear, showing the text *insert error text*

Logging out

When pressing the Log out button, the user will no longer be able to submit new events. The Log out button will only be visible is the user is currently logged in to the system.

Submiting event

Eiffel data

In the field Eiffel data, the user enters the Eiffel data the user wants to submit. The data must follow the protocol which determines how an Eiffel event is structured.

Look up data

If the user wants to link to another Eiffel event but does not know the link target. The user will have to provide a separate look up object.

Send to message bus

Is the user checks the Send to message bus checkbox the message will be sent to the message bus.

Edition

The user needs enter which edition the event should be written by, this is done in the field titled Edition.

Error messages

If a user enters an incorrect event an error message will appear.
