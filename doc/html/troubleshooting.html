Docker tips, tricks and troubleshooting

MongoDB with Docker

  • You can access the MongoDB running in the Docker image through the terminal. To enter the image "OS" run: exec -it mongo /bin/bash

  • To access the database run: mongo

You can now use MongoDB commands like you normally would, show dbs, db.dropDatabase() etc.

To enter the database "eventDB" that contains all the events write use eventDB.

To show what type of events that exist in the database write show collections.

To display the json-object that actually exists in the database write eg db.EiffelArtifactCreatedEvent.find({}).

To drop the whole database write db.dropDatabase().

Troubleshooting Docker for Windows

If you receive the following error:

ERROR: for mongo Cannot create container for service mongo: b'Drive has not been shared'

Follow these steps to solve it:
  1. Type the following in cmd/Powershell set COMPOSE_CONVERT_WINDOWS_PATHS=1
  2. Restart Docker
  3. Go to Settings in Docker->Shared Drives->-Reset credentials->-Select drive->Apply
